# Chapter 3: Software Development Life Cycle (SDLC)

## Learning Objectives

After completing this chapter, you will be able to:

* Understand what SDLC is.
* Explain each phase of the Software Development Life Cycle.
* Understand the role of QA in each phase.
* Learn the importance of following SDLC.
* Answer common SDLC interview questions.

---

# 1. What is SDLC?

Software Development Life Cycle (SDLC) is a structured process followed by software development teams to design, develop, test, deploy, and maintain software applications.

The primary objective of SDLC is to deliver high-quality software within the estimated time and budget while meeting customer requirements.

---

# 2. Why Do We Need SDLC?

SDLC provides a systematic approach to software development.

It helps to:

* Deliver high-quality software.
* Reduce development costs.
* Complete projects within the planned timeline.
* Improve communication among team members.
* Minimize project risks.
* Increase customer satisfaction.

---

# 3. Phases of SDLC

The Software Development Life Cycle consists of the following phases:

1. Requirement Gathering
2. Planning
3. Design
4. Development
5. Testing
6. Deployment
7. Maintenance

---

# 4. Requirement Gathering

In this phase, business requirements are collected from the client or stakeholders.

### Activities

* Understand customer requirements.
* Discuss business goals.
* Prepare the Software Requirement Specification (SRS).

### Participants

* Client
* Business Analyst (BA)
* Product Owner
* Project Manager

### Deliverable

Software Requirement Specification (SRS)

---

# 5. Planning

After gathering requirements, the project team prepares the project plan.

### Activities

* Cost estimation
* Time estimation
* Resource planning
* Risk analysis

### Participants

* Project Manager
* Technical Lead
* Management Team

### Deliverables

* Project Plan
* Cost Estimation
* Resource Plan

---

# 6. Design

The design team prepares the architecture of the application.

### Activities

* Database Design
* UI/UX Design
* High-Level Design (HLD)
* Low-Level Design (LLD)

### Participants

* Solution Architect
* Technical Lead
* UI/UX Designer

### Deliverables

* HLD Document
* LLD Document
* UI Designs

---

# 7. Development

Developers begin coding the application based on the approved design.

### Activities

* Writing source code
* Unit Testing
* Code Reviews

### Participants

* Developers

### Deliverable

Application Source Code

---

# 8. Testing

The QA team verifies whether the application works according to the requirements.

### Activities

* Test Planning
* Test Scenario Preparation
* Test Case Writing
* Test Execution
* Bug Reporting
* Regression Testing
* Test Summary Report

### Participants

* QA Engineers
* Test Lead

### Deliverables

* Test Cases
* Bug Reports
* Test Summary Report
* Requirement Traceability Matrix (RTM)

---

# 9. Deployment

Once testing is completed successfully, the application is released to the production environment.

### Activities

* Production Deployment
* Smoke Testing
* Production Verification

### Participants

* DevOps Team
* Release Manager
* Operations Team

---

# 10. Maintenance

After deployment, the application is continuously monitored and maintained.

### Activities

* Bug Fixes
* Performance Improvements
* Security Updates
* Feature Enhancements

Maintenance continues until the application reaches the end of its lifecycle.

---

# 11. SDLC Workflow

```text
Requirement Gathering
        ↓
Planning
        ↓
Design
        ↓
Development
        ↓
Testing
        ↓
Deployment
        ↓
Maintenance
```

---

# 12. Role of a QA Engineer in SDLC

A QA Engineer contributes throughout the SDLC by:

* Reviewing requirements.
* Identifying missing or unclear requirements.
* Preparing test scenarios.
* Writing test cases.
* Executing test cases.
* Reporting defects.
* Performing regression testing.
* Preparing test summary reports.
* Supporting User Acceptance Testing (UAT).

---

# 13. Real-World Example

Suppose a company wants to develop an Online Food Delivery Application.

### Requirement Gathering

The client requests features such as:

* User Registration
* Login
* Restaurant Search
* Add to Cart
* Online Payment
* Order Tracking

### Planning

The project manager estimates:

* Timeline: 6 Months
* Team: 10 Developers and 4 QA Engineers
* Budget: Based on project scope

### Design

The architecture, database, and user interface are designed.

### Development

Developers build all the required features.

### Testing

QA Engineers verify:

* Registration
* Login
* Search
* Cart
* Payment
* Order Tracking

### Deployment

The application is released to production.

### Maintenance

Future updates include:

* Coupons
* Referral System
* Dark Mode
* Performance Improvements

---

# 14. Advantages of SDLC

* Provides a structured development process.
* Improves software quality.
* Reduces project risks.
* Better project planning.
* Easier project management.
* Improves customer satisfaction.
* Helps deliver projects on time.

---

# 15. Limitations of SDLC

* Changes in requirements can impact the project.
* Documentation requires additional effort.
* Initial planning may take more time.
* Large projects require coordination between multiple teams.

---

# Interview Questions

### 1. What is SDLC?

Software Development Life Cycle (SDLC) is a structured process used to develop, test, deploy, and maintain high-quality software applications.

---

### 2. What are the phases of SDLC?

* Requirement Gathering
* Planning
* Design
* Development
* Testing
* Deployment
* Maintenance

---

### 3. What is the role of QA in SDLC?

QA Engineers participate in requirement analysis, test planning, test case creation, test execution, defect reporting, regression testing, and release validation.

---

### 4. What is the output of the Requirement Gathering phase?

The main output is the Software Requirement Specification (SRS) document.

---

### 5. Which phase comes after Development?

Testing.

---

# Key Takeaways

* SDLC is a structured process for developing software.
* It consists of seven phases.
* Each phase has specific activities and deliverables.
* QA Engineers play an important role throughout the SDLC.
* Following SDLC helps deliver high-quality software on time and within budget.
