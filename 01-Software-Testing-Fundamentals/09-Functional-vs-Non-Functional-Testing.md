# Chapter 9: Functional Testing vs Non-Functional Testing

## Learning Objectives

After completing this chapter, you will be able to:

* Understand Functional Testing.
* Understand Non-Functional Testing.
* Differentiate between Functional and Non-Functional Testing.
* Learn the different types of Functional and Non-Functional Testing.
* Answer common interview questions.

---

# 1. What is Functional Testing?

Functional Testing is a type of software testing that verifies whether the application's features and functionalities work according to the specified business requirements.

It focuses on **"What the application does."**

---

# 2. Objectives of Functional Testing

* Verify business requirements.
* Validate application functionality.
* Ensure expected results are achieved.
* Detect functional defects.

---

# 3. Examples of Functional Testing

Suppose you are testing an Online Shopping Application.

You verify:

* User Registration
* Login
* Search Product
* Add to Cart
* Remove from Cart
* Payment
* Order Confirmation
* Logout

If all these features work correctly, the application passes Functional Testing.

---

# 4. Types of Functional Testing

* Unit Testing
* Integration Testing
* System Testing
* Smoke Testing
* Sanity Testing
* Regression Testing
* Re-testing
* User Acceptance Testing (UAT)

---

# 5. Advantages of Functional Testing

* Ensures business requirements are met.
* Detects functional defects.
* Improves software quality.
* Increases customer satisfaction.

---

# 6. Limitations of Functional Testing

* Does not measure performance.
* Does not check security.
* Does not evaluate usability.
* Cannot determine system scalability.

---

# 7. What is Non-Functional Testing?

Non-Functional Testing verifies how well the application performs rather than what it does.

It focuses on quality attributes such as performance, security, usability, and reliability.

---

# 8. Objectives of Non-Functional Testing

* Measure application performance.
* Verify security.
* Check usability.
* Evaluate scalability.
* Ensure reliability.

---

# 9. Types of Non-Functional Testing

* Performance Testing
* Load Testing
* Stress Testing
* Volume Testing
* Scalability Testing
* Security Testing
* Usability Testing
* Compatibility Testing
* Reliability Testing
* Recovery Testing

---

# 10. Examples of Non-Functional Testing

For an Online Shopping Application:

* Can 10,000 users log in simultaneously?
* How quickly does the application load?
* Is user data secure?
* Does the application work on Chrome, Firefox, Safari, and Edge?
* Does the application work on Android and iOS?

---

# 11. Difference Between Functional and Non-Functional Testing

| Functional Testing | Non-Functional Testing |
|-------------------|------------------------|
| Verifies application functionality | Verifies application quality |
| Focuses on business requirements | Focuses on performance and reliability |
| Tests features | Tests quality attributes |
| Example: Login | Example: Login response time |
| Performed before release | Usually performed after functional stability |

---

# 12. Real-Time Example

Suppose your company develops an Online Banking Application.

### Functional Testing

QA verifies:

* Login
* Fund Transfer
* Account Balance
* Transaction History
* Bill Payments

### Non-Functional Testing

QA verifies:

* Response time
* Server performance
* Security
* Browser compatibility
* Mobile responsiveness

---

# 13. Best Practices

* Complete Functional Testing before starting Non-Functional Testing.
* Prepare separate test cases for both testing types.
* Use automation where appropriate.
* Document all test results.
* Test on multiple environments.

---

# 14. Common Mistakes

* Ignoring Non-Functional Testing.
* Confusing functionality with performance.
* Not testing different browsers and devices.
* Assuming Functional Testing alone is enough.

---

# Interview Questions

### 1. What is Functional Testing?

Functional Testing verifies that software features work according to business requirements.

---

### 2. What is Non-Functional Testing?

Non-Functional Testing verifies the application's performance, security, usability, and other quality attributes.

---

### 3. Give examples of Functional Testing.

* Login
* Registration
* Search
* Payment
* Logout

---

### 4. Give examples of Non-Functional Testing.

* Performance Testing
* Security Testing
* Load Testing
* Stress Testing
* Compatibility Testing

---

### 5. Can an application pass Functional Testing but fail Non-Functional Testing?

Yes.

An application may have all features working correctly but still fail due to poor performance, security issues, or compatibility problems.

---

# Key Takeaways

* Functional Testing verifies **what** the application does.
* Non-Functional Testing verifies **how well** the application performs.
* Both testing types are essential for delivering high-quality software.
* Functional and Non-Functional Testing complement each other.
